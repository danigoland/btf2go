## New friction (if any)

- **FRICTION-1 (medium)**: `clang -target bpf` with `#include <linux/bpf.h>` fails — `asm/types.h` not found.
  This is the pre-existing `asm/types.h` friction documented in the aya-quickstart.md "Common pitfalls" section.
  M2.7 worked around it by removing the include and using bare typedefs. Aligns with the quickstart note.
  Not new; documented.

- **FRICTION-2 (high)**: Plain `clang -O2 -g -target bpf -c` does NOT emit `.BTF` section even with `-g`.
  M2.7 saw the PR #66 loud error from `btf2go inspect` and understood the cause ("bpf-linker version mismatch").
  However, M2.7 then tried `bpf-linker --btf` which panicked (LLVM shared lib mismatch on this VM).
  Root cause: bpf-linker v0.10.3 in the template requires `libLLVM-22-rust-nightly.so` but the VM only has
  LLVM-19. Previously documented in PR #62 (Kimi run). Not new; pre-existing infra limitation.

- **FRICTION-3 (medium)**: `bpf-linker --btf` panics — `unable to find LLVM shared lib`. Known from PR #62.

- **FRICTION-4 (low)**: Wrong `--type` name (`event_t` vs `events_t`) when using test fixture.
  The `btf2go generate` error message included `Did you mean: [events_t]?` which correctly guided recovery.

- **FRICTION-5 (low)**: Go import issues (relative paths not allowed in module mode, `.c` file in module dir).
  Standard Go gotchas, not btf2go-specific.

### net-new findings (not previously tracked)

1. **bpf-linker LLVM mismatch in VM template** (pre-existing since PR #62): bpf-linker v0.10.3 requires
   LLVM-22-nightly but VM template ships LLVM-19. Already known. M2.7 handled it gracefully by falling back

2. **`aya_build::build()` in aya-quickstart.md is wrong API**: The quickstart's `build.rs` note says
   `aya_build::build().unwrap()` but aya-build v0.1.3 exposes `build_ebpf()`. This is a new issue
   introduced or exposed by PR #64's note. However, the `.cargo/config.toml` `rustflags = ["-C",
   "link-arg=--btf"]` path works without a `build.rs` entirely (confirmed in pre-run testing). Low priority.

3. **`go install @latest` gets v0.3.1** (predates all fixes): When `go install
   github.com/danigoland/btf2go/cmd/btf2go@latest` was run, it resolved v0.3.1. The fixes are on master
   but not yet tagged. Any user following `@latest` from the README would get the unfixed version.
   **Recommend tagging v0.3.2 to ship the 3 fixes.**
